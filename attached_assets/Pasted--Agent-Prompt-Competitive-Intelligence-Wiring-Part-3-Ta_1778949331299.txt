# Agent Prompt: Competitive Intelligence Wiring — Part 3
# Target repo: nsd-platform-shell (Next.js dashboard)

## Context

Parts 1 and 2 are complete:
- **nsd-ods-api** exposes 7 competitive intelligence REST endpoints (live in Supabase/Replit)
- **nsd-integrations** drives weekly Apify crawls and posts results to those endpoints

Your job is to build the `/dashboard/competitive` page in nsd-platform-shell.

---

## Data sources (all ODS API)

Base URL stored in `NSD_ODS_API_BASE_URL` env var. All requests are server-side
(Next.js Route Handlers or `getServerSideProps`) using `NSD_ODS_API_SERVICE_TOKEN`
as the Bearer token. Never expose the token to the browser.

| Endpoint | Used for |
|----------|----------|
| `GET /api/v1/competitive/summary` | Header stat cards |
| `GET /api/v1/competitive/weekly-insights/latest` | Weekly insight panel |
| `GET /api/v1/competitive/pages/changes?limit=50` | Change feed table |

### GET /api/v1/competitive/summary — response shape
```json
{
  "success": true,
  "data": {
    "competitors_tracked": 10,
```

Query params supported: `limit`, `offset`, `competitor` (domain), `page_type`, `change_type` (`new` | `changed`).

## Component breakdown

### `app/dashboard/competitive/page.tsx`
Server component. Fetches all 3 endpoints in parallel (`Promise.all`).
Passes data as props to client components.

```typescript
// Pseudocode — adapt to the repo's data fetching pattern
const [summary, insight, changes] = await Promise.all([
  fetchOds('/api/v1/competitive/summary'),
  fetchOds('/api/v1/competitive/weekly-insights/latest'),
  fetchOds('/api/v1/competitive/pages/changes?limit=50'),
]);
```

### `components/competitive/SummaryCards.tsx`
Five stat cards (competitors_tracked, pages_tracked, this_week_new, this_week_changed,
competitors_with_changes). Use the existing stat card pattern from other dashboard pages.

### `components/competitive/WeeklyInsightPanel.tsx`
Priority badge (`HIGH` = red, `MEDIUM` = amber, `LOW` = green).
Summary paragraph. Key changes table (10 rows max). Recommended actions list with priority
badge per row. Renders a "No insight yet" empty state when `data` is null.

### `components/competitive/ChangeFeed.tsx`
Client component (uses `useState` for filters + pagination). Calls the ODS endpoint from
a Next.js proxy Route Handler (never directly from browser). Columns:
- Competitor name + domain badge
- Page URL (truncated, links to the live URL)
- Page type badge (colour-coded: pricing=red, customizer=purple, product=blue, blog=grey, other=slate)
- Change type badge (`new` = green, `changed` = amber)
- Buyer segment pill
- Crawled at (relative time, e.g. "3 days ago")

Filters: change_type (All / New / Changed), competitor domain (dropdown from the tracked
competitor list), page_type (All / product / pricing / customizer / blog / industry_page / other).

Pagination: show 50 at a time, "Load more" button increments offset.

## Next.js proxy Route Handler
Create `app/api/competitive/changes/route.ts` (or equivalent) so the browser calls go through
Next.js, not directly to the ODS. This keeps `NSD_ODS_API_SERVICE_TOKEN` server-side only.

```typescript
// app/api/competitive/changes/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(req: NextRequest) {
  const { searchParams } = req.nextUrl;
  const qs = searchParams.toString();

  const res = await fetch(
    `${process.env.NSD_ODS_API_BASE_URL}/api/v1/competitive/pages/changes?${qs}`,
    {
      headers: {
        Authorization: `Bearer ${process.env.NSD_ODS_API_SERVICE_TOKEN}`,
      },
      cache: 'no-store',
    }
  );

  const data = await res.json();
  return NextResponse.json(data, { status: res.status });
}
```
