PHASE C — REPLACE GENERIC RECOMMENDATION COPY WITH DETERMINISTIC TEMPLATES
==================================================

Recommendation copy must now be generated from deterministic templates based on `primary_remedy`, opportunity family, and evidence.

Do NOT use generic text like:
- improve this page
- optimize content
- add more internal links
- fix SEO issues

Instead, implement structured templates by remedy family.

At minimum support these recommendation templates:

### 1. create_new_page
Example pattern:
Create a dedicated **[cluster/topic]** page because competitors are ranking for this topic and NSD does not have a strong page presence. Demand is supported by **Ahrefs volume [X]** and **[CPC if available]**.

### 2. strengthen_existing_page
Strengthen **[page/query]** because NSD already has a page and some organic visibility, but competitors are outperforming it. Current organic signal is **[GSC metrics]** and the opportunity is supported by **[external evidence]**.

### 3. add_internal_links
Add internal links to **[target page]** because NSD has ranking potential but appears under-supported internally. Internal-link signal is **[strong/moderate/position_only]** and the current ranking page is **[URL]**.

### 4. pursue_backlinks
Prioritize backlink outreach for **[cluster/page]** because competitors benefit from stronger authority support. The strongest current signal includes **[referring domain / DR / competitor]**.

### 5. maintain_paid_support
Maintain paid support for **[query/cluster]** because paid demand is active while SEO is still weak or immature. Paid signal shows **[cost / conversions / CPC]**.

### 6. metadata_ctr_optimization
Improve metadata for **[page/query]** because NSD already ranks within visible range but CTR is weak relative to impressions. Current signal is **[impressions / position / CTR]**.

### 7. hybrid
Use a deterministic combined template based on the dominant paired remedies, for example:
- strengthen_existing_page + pursue_backlinks
- create_new_page + pursue_backlinks
- create_new_page + maintain_paid_support

Requirements:
- recommendation text must be deterministic
- recommendation text must use real evidence fields
- no vague placeholders
- no AI generation in this phase
- suppress or exclude branded/no-action rows completely
