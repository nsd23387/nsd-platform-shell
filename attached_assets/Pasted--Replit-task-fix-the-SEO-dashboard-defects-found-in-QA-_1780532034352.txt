- Build a **Recommendations list page** (e.g. `/dashboard/seo/recommendations`) listing **all gate-accepted, pending recommendations** (`seo_execution_candidate WHERE gate_status='accepted' AND approval_status='pending'`, canonical_live targets), **paginated** to the full set.
- **Filter + sort** by: target page URL (free-text, e.g. "for-businesses"), **mutation_type** (title / meta / h1 / schema / internal_link / redirect), status, and **opportunity_score** (default sort desc). Reuse the **Competitors table pattern** — it already does filter + full-list well.
- Each row shows: rank/score, type tag, target page, one-line `evidence_summary`, and links to its **Action Card** (`/dashboard/seo/action/[candidate_id]`, which works). Inline Approve/Defer optional.
- Repoint **"View all N recommendations →"** (Command Center) to this page.
- **Acceptance:** "View all" lands on a list of all ~447; filtering to "/for-businesses/" or to "title" narrows correctly; each row opens its Action Card; counts match the DB.

### D2 — Performance page cards must open the page's dossier
The subtitle says "Click any page to open its dossier," but clicking a card does nothing (verified on `/for-businesses/` via its element ref — no navigation/drawer).
- Wire each Performance card to open that page's dossier/recommendations (route to a page dossier view, or to the Recommendations list pre-filtered to that URL).

### D3 — "Review top N" batch-approve button is dead

- Implement the review-then-confirm modal: list the top N pending high-confidence candidates with **per-item deselect**, then **Approve & Queue** writes `approval_status` via `lib/seoApi` (draft-only; nothing publishes). Cancel closes with no change.
