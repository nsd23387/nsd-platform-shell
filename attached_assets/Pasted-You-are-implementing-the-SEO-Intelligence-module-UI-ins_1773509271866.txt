You are implementing the **SEO Intelligence module UI** inside the `nsd-platform-shell` repository.

Follow these instructions exactly.

The UI will consume the cluster engine APIs from `nsd-ods-api`.

The UI must remain **read-only except for approval actions**.

Do NOT compute SEO metrics in the UI.

All SEO logic must remain in `nsd-ods-api`.

---

# OBJECTIVE

Create a new **SEO Intelligence module** inside Platform Shell that allows management to:

• view keyword clusters
• see cluster opportunities
• review SEO recommendations
• approve or reject recommendations
• view learning outcomes

This module will power the **SEO decision workflow**.

---

Add routes:

/seo
/seo/clusters
/seo/opportunities
/seo/recommendations
/seo/outcomes

# STEP 3 — Create API Client

Create file:

src/api/seoApi.ts

Functions:

getClusters()

GET /api/v1/seo/clusters

getClusterMembers(clusterId)

GET /api/v1/seo/clusters/:id

getClusterOpportunities()

GET /api/v1/seo/cluster-opportunities

getRecommendations()

GET /api/v1/seo/recommendations

approveRecommendation(id)

POST /api/v1/seo/recommendations/:id/approve

rejectRecommendation(id)

POST /api/v1/seo/recommendations/:id/reject

All requests must include auth token and enforce permission:

seo:read

Approval endpoints require:

seo:approve
