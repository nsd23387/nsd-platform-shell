## Non-negotiable data & governance rules (do not regress these — the wiring already does most of this correctly)
1. **Only `status_class='canonical_live'` pages are ever shown as optimization targets.** `lost` → restore/redirect queue only; `excluded` → never shown; `pending_verification` → only with a "verify" flag. Read everything through `analytics.seo_page_inventory` (the Page Universe). No screen may read a legacy table that bypasses the inventory.
2. **Position = top-query average position, NOT best-ever position.** The portfolio API currently returns `gsc_best_position` (all-time min = 1 for the homepage) and buckets off it — this is a bug. The homepage ranks ~46 for "neon signs," so it must read as **strategic**, not "win." Use this validated query (it runs clean on prod) for the per-page top query + its avg position (`top_q_pos`):
   ```sql
   WITH pq AS (SELECT regexp_replace(page_url,'^https?://[^/]+','') path, query,
                      sum(impressions) qi, round(avg(avg_position),1) qp
               FROM analytics.metrics_search_console_query_page_daily GROUP BY 1,2),
   top AS (SELECT DISTINCT ON (path) path, query topq, qi, round(qp,0) qpos
           FROM pq ORDER BY path, qi DESC)
   SELECT i.url, i.content_type, i.status_class, i.gsc_impressions,
          t.topq AS top_query, t.qi::int AS top_q_impr, t.qpos::int AS top_q_pos,
          e.search_volume AS kw_volume, e.cpc::numeric(10,2) AS kw_cpc,
          e.keyword_difficulty::int AS kw_difficulty, e.metrics_source
   FROM analytics.seo_page_inventory i
   LEFT JOIN top t ON t.path = regexp_replace(i.url,'^https?://[^/]+','')
   LEFT JOIN analytics.external_query_intelligence e ON e.normalized_query =
     TRIM(BOTH FROM regexp_replace(lower(regexp_replace(regexp_replace(t.topq,'[^a-zA-Z0-9 ]','','g'),'\y(for|the|and|a)\y','','gi')),'\s+',' ','g'))
   WHERE i.status_class IN ('canonical_live','lost')
   ORDER BY i.gsc_impressions DESC NULLS LAST;
   ```
   Buckets: `lost`=status lost · `win`=canonical_live AND `top_q_pos` 1–30 · `strategic`=canonical_live AND (`top_q_pos`>30 OR null) · `fix`=trailing-slash/dupe of another canonical_live URL.
3. **Draft-only actuation.** Approve/Defer/Modify write to `seo_execution_candidate` via `lib/seoApi` — never publish to WordPress directly, never auto-approve. No inline POST in pages.
4. **Honest empty/pending states.** Show verified-vs-pending explicitly; if a metric is missing say so (e.g. "no DataForSEO volume"), don't fabricate.
