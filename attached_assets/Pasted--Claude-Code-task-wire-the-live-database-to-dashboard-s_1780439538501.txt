## Governance rule (non-negotiable)
Every page surfaced as an optimization target MUST be `status_class='canonical_live'`. `lost` → its own restore/redirect queue; `excluded` → hidden; `pending_verification` → shown only with a "verify" flag, never as a confident target. No 404s as opportunities.

## 1. Proxy read routes (app/api/proxy/seo/…, Node pg, read-only)

### `GET /api/proxy/seo/portfolio` — Level 1 (page portfolio)
Returns canonical_live + lost pages with demand + keyword value + bucket. Use this **validated** query (it ran clean on prod):
```sql
WITH pq AS (SELECT regexp_replace(page_url,'^https?://[^/]+','') path, query, sum(impressions) qi, round(avg(avg_position),1) qp
            FROM analytics.metrics_search_console_query_page_daily GROUP BY 1,2),
top AS (SELECT DISTINCT ON (path) path, query topq, qi, round(qp,0) qpos FROM pq ORDER BY path, qi DESC)
SELECT i.url, i.content_type, i.status_class, i.gsc_impressions, i.gsc_best_position,
       t.topq AS top_query, t.qi::int AS top_q_impr, t.qpos::int AS top_q_pos,
       e.search_volume AS sv, e.cpc::numeric(10,2) AS cpc, e.keyword_difficulty::int AS kd, e.metrics_source
FROM analytics.seo_page_inventory i
LEFT JOIN top t ON t.path = regexp_replace(i.url,'^https?://[^/]+','')
LEFT JOIN analytics.external_query_intelligence e ON e.normalized_query =
  TRIM(BOTH FROM regexp_replace(lower(regexp_replace(regexp_replace(t.topq,'[^a-zA-Z0-9 ]','','g'),'\y(for|the|and|a)\y','','gi')),'\s+',' ','g'))
WHERE i.status_class IN ('canonical_live','lost')
ORDER BY i.gsc_impressions DESC NULLS LAST;
```
Bucket each row server- or client-side: `lost` (status_class='lost') · `win` (canonical_live AND top_q_pos BETWEEN 1 AND 30) · `strategic` (canonical_live AND top_q_pos > 30 or null) · `fix` (trailing-slash/duplicate of another canonical_live URL).

### `GET /api/proxy/seo/page?url=` — Level 2 (one page's dossier)
For the page: its top ~8 queries with demand + keyword value (join external_query_intelligence as above, parameterized by url — use a parameterized query, not string interpolation), its `seo_page_inventory` row, and its **gate-accepted candidates** from `seo_execution_candidate WHERE target_page_url=$1 AND gate_status='accepted'` (the real Lane-1 actions with evidence).

### `GET /api/proxy/seo/suppressed` — keep existing (gate-suppressed audit).

## 2. Review UI (app/dashboard/seo) — render the dossier model
Match the prototype / design spec. Two levels:
- **Level 1 portfolio:** pages grouped Lost / Win-now / Strategic / Fix. Card = URL + status pill + top query · impressions · position + keyword value line (`vol · KD · $cpc` from DataForSEO, or "no DataForSEO volume") + the one-line read. KPI strip on top (counts per bucket + lost impressions). Clicking a card → Level 2.
- **Level 2 page dossier:** demand table (queries with real/discard read — flag competitor-brand/foreign/repair as discard), keyword target (primary), and **Recommended actions routed by lane**:
  - **Lane 1 (engine drafts → approve):** the page's `seo_execution_candidate` gate-accepted rows (real), each with Approve/Reject via `lib/seoApi`.
  - **Lane 2 (you execute in Rank Math):** lost-page redirect / canonical / noindex (derive from status_class + bucket).
  - **Lane 3 (hand to SEO/PR):** off-page brief for authority-bound strategic pages (target keyword + position + why).
- **Lost** pages render in their own queue (restore/redirect, intent-validated — never auto slug-match). **Excluded** hidden. **pending_verification** shown with a "verify live" flag.
