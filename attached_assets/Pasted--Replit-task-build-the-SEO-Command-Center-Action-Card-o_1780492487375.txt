## Non-negotiable rules (don't regress)
1. **Only `status_class='canonical_live'` pages are optimization targets.** `lost` → restore/redirect queue; `excluded` → never shown; `pending_verification` → only with a "verify" badge. Filter through `seo_page_inventory`.
2. **Position = the top query's average position, NOT `gsc_best_position`.** The earlier UI showed `gsc_best_position` (all-time min = 1) and bucketed off it, mislabeling the homepage a "win" when "neon signs" ranks ~46. Use the **top query's `position` from `seo_page_dossier.demand`** (the highest-impression query), or compute `top_q_pos` via the validated query below. Bucket off that.
3. **Draft-only.** Approve/Defer/Reject write to `seo_execution_candidate.approval_status` via `lib/seoApi` — never publish to WordPress, never auto-approve. No inline POST in pages.
4. **One backing source per screen; honest empty/pending states** (verified-vs-pending explicit; "no DataForSEO volume" when missing).

Validated `top_q_pos` query (use if not taking it from `demand[0]`):
```sql
WITH pq AS (SELECT regexp_replace(page_url,'^https?://[^/]+','') path, query,
                   sum(impressions) qi, round(avg(avg_position),1) qp
            FROM analytics.metrics_search_console_query_page_daily GROUP BY 1,2),
top AS (SELECT DISTINCT ON (path) path, query topq, qi, round(qp,0) qpos
        FROM pq ORDER BY path, qi DESC)
SELECT regexp_replace(i.url,'^https?://[^/]+','') path, t.topq top_query, t.qpos top_q_pos
FROM analytics.seo_page_inventory i
LEFT JOIN top t ON t.path = regexp_replace(i.url,'^https?://[^/]+','')
WHERE i.status_class IN ('canonical_live','lost');
```

## Proxy read routes (`app/api/proxy/seo/…`, Node pg, read-only, parameterized)
- `GET /portfolio` — Level-1 page list: `seo_page_inventory` (canonical_live + lost) joined to `seo_page_dossier.priority` + top-query/`top_q_pos` + keyword value. Bucket: `lost` (status lost) · `win` (canonical_live & top_q_pos 1–30) · `strategic` (canonical_live & top_q_pos >30 or null) · `fix` (trailing-slash/dupe).
- `GET /queue` — the **Detections / "DO THIS NEXT"** list: `seo_execution_candidate WHERE gate_status='accepted' AND approval_status='pending'`, joined to inventory (canonical_live only) and dossier, ranked by `opportunity_score` desc. Include `mutation_type, target_page_url, evidence_summary, opportunity_score, confidence_tier, regate_review_flag`.
- `GET /page?url=` — Level-2 dossier: the `seo_page_dossier` row (demand, keyword_targets, routed_queries, state, ranked_actions) + that page's accepted `seo_execution_candidate` rows (for Approve/Reject + `evidence` + `current_value_snapshot`/`proposed_value`) + recent `seo_gate_transition` rows for those candidates.
- `GET /offpage` — `seo_offpage_brief` (Lane-3 list).
- `GET /suppressed` — gate-suppressed audit (existing); group by `gate_reasons`.
