Mission:
Make metadata CTR recommendations actionable in the UI by turning them from passive diagnostics into operator-ready execution items.

Problem to solve:
Today, a metadata CTR recommendation explains:
- the cluster
- the issue
- the score
- the measurement plan

But it does NOT tell the operator:
- which exact page to edit
- what the current SEO title is
- what the current meta description is
- what the proposed replacement title/meta should be
- what exact action to take next

That makes the recommendation understandable only to an SEO specialist and too slow for execution.

Goal:
For Phase-1 metadata CTR recommendations, expose a clear action layer in the existing Recommendations detail view so an operator can immediately understand:
1. the target page
2. the current metadata
3. the proposed metadata
4. the rationale
5. the next action to take

Critical policy:
- Do NOT redesign the SEO UI.
- Do NOT add a new page.
- Reuse the existing Recommendations page and detail panel.
- Keep changes additive and narrow.
- Do NOT build autonomous apply-to-site behavior in this task.
- Focus on actionability, not full automation.

When a recommendation like:
- `metadata_ctr_optimization`

the operator should be able to open the detail panel and immediately see:
- target page URL, e.g. `/for-businesses/`
- current title tag
- current meta description
- suggested title tag
- suggested meta description
- why the new version is better
- an action CTA such as:
  - Approve
  - Copy metadata
  - Create execution task
